## Core Technique: Circular Array

A circular array uses two pointers (`start`, `end`) plus the modulo operator `%` to turn a flat array into a logical ring. This lets you add/remove at **both ends in O(1)** — the one operation a normal dynamic array can't do cheaply.

**When to use:** Implementing a deque, sliding-window buffer, or any structure that needs O(1) head insertions/deletions.

**Why use it:** Normal arrays cost O(N) to add/remove at the head (element shift). Circular arrays avoid the shift entirely by just moving a pointer.

**Key convention:** `start` is **closed** (points to the first valid element), `end` is **open** (points one past the last). Interval is `[start, end)`.

**Template:**

```python
class CycleArray:
    def __init__(self, size=1):
        self.arr = [None] * size
        self.size = size
        self.start = 0   # closed — first valid element
        self.end = 0     # open   — one past last valid element
        self.count = 0

    def resize(self, new_size):
        new_arr = [None] * new_size
        for i in range(self.count):
            new_arr[i] = self.arr[(self.start + i) % self.size]
        self.arr = new_arr
        self.start = 0
        self.end = self.count
        self.size = new_size

    # start is closed → move pointer first, then write
    def add_first(self, val):
        if self.is_full(): self.resize(self.size * 2)
        self.start = (self.start - 1 + self.size) % self.size
        self.arr[self.start] = val
        self.count += 1

    # start is closed → clear first, then advance pointer
    def remove_first(self):
        if self.is_empty(): raise Exception("empty")
        self.arr[self.start] = None
        self.start = (self.start + 1) % self.size
        self.count -= 1
        if self.count > 0 and self.count == self.size // 4:
            self.resize(self.size // 2)

    # end is open → write first, then advance pointer
    def add_last(self, val):
        if self.is_full(): self.resize(self.size * 2)
        self.arr[self.end] = val
        self.end = (self.end + 1) % self.size
        self.count += 1

    # end is open → retreat pointer first, then clear
    def remove_last(self):
        if self.is_empty(): raise Exception("empty")
        self.end = (self.end - 1 + self.size) % self.size
        self.arr[self.end] = None
        self.count -= 1
        if self.count > 0 and self.count == self.size // 4:
            self.resize(self.size // 2)

    def get_first(self):
        if self.is_empty(): raise Exception("empty")
        return self.arr[self.start]

    def get_last(self):
        if self.is_empty(): raise Exception("empty")
        return self.arr[(self.end - 1 + self.size) % self.size]

    def is_full(self):  return self.count == self.size
    def is_empty(self): return self.count == 0
```

## Why the Half-Open Interval `[start, end)`

Initializing `start = end = 0` gives an empty interval with no special-casing needed. Moving `end` right by one immediately captures one element. A fully-closed `[start, end]` starts with one element already "in" the array; a fully-open `(start, end)` still has zero elements after advancing `end` once — both require awkward edge-case handling.

**The pointer rules follow directly from the interval choice:**

| Operation | Rule | Reason |
|---|---|---|
| `add_first` | move `start` left, then write | `start` must point at the element after writing |
| `remove_first` | clear, then move `start` right | `start` currently points at a valid element |
| `add_last` | write at `end`, then move right | `end` is the next empty slot |
| `remove_last` | move `end` left, then clear | must step back to reach the last element |

---

## Key Patterns to Remember

| Operation | Circular Array | Plain Array |
|---|---|---|
| Add/remove at tail | O(1) | O(1) amortized |
| Add/remove at head | **O(1)** | O(N) — element shift |
| Random access by index | O(1) via `(start + i) % size` | O(1) direct |
| Insert/delete at index k | O(N) — still needs shift | O(N) |

The circular array strictly dominates the plain array for head operations and matches it everywhere else. Yet most standard-library dynamic arrays (`list` in Python, `ArrayList` in Java) don't use it — because random-index access via `(start + i) % size` adds a modulo on every read, which is a constant-factor slowdown that matters in practice for tight loops.
