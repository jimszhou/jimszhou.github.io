## Core Technique: Linked List Manipulation + Heap

Today's problems combine several linked list patterns: **dummy-node pointer threading** for safe deletion/construction, **digit-by-digit arithmetic with carry**, and **Floyd's cycle detection on implicit graphs**. A min-heap also appears for multi-way ordered enumeration.

```python
# Dummy node + prev pointer for safe deletion
dummy = ListNode(0, head)
prev = dummy
while prev.next:
    if should_remove(prev.next):
        prev.next = prev.next.next
    else:
        prev = prev.next
return dummy.next
```

**When to use:** Removing nodes in-place, building result lists from digit arithmetic, or detecting cycles in value-index mappings.

**Why:** O(n) time, O(1) space for most linked list manipulations. Heap gives O(k log k) for bounded enumeration.

## Problems Studied

### 82. Remove Duplicates from Sorted List II

Delete all nodes that have duplicate values — not just extra copies, but every occurrence. Use a `prev` pointer behind a scan pointer; when duplicates are found, skip the entire group.

**Prior knowledge:** [83. Remove Duplicates from Sorted List](https://leetcode.com/problems/remove-duplicates-from-sorted-list/) — the simpler version that keeps one copy of each duplicate.

```python
def deleteDuplicates(head):
    dummy = ListNode(0, head)
    prev = dummy
    cur = head
    while cur:
        if cur.next and cur.val == cur.next.val:
            while cur.next and cur.val == cur.next.val:
                cur = cur.next
            prev.next = cur.next
        else:
            prev = prev.next
        cur = cur.next
    return dummy.next
```

### 373. Find K Pairs with Smallest Sums

Given two sorted arrays, find k pairs with smallest sums. Treat it like a k-way merge: each row `(nums1[i], nums2[0..])` is a sorted stream. Push initial pairs, then expand by incrementing the `nums2` index.

**Prior knowledge:** [23. Merge k Sorted Lists](https://leetcode.com/problems/merge-k-sorted-lists/) — same heap-based k-way merge pattern. Also [378. Kth Smallest Element in a Sorted Matrix](https://leetcode.com/problems/kth-smallest-element-in-a-sorted-matrix/).

```python
import heapq
def kSmallestPairs(nums1, nums2, k):
    if not nums1 or not nums2:
        return []
    heap = [(nums1[i] + nums2[0], i, 0) for i in range(min(k, len(nums1)))]
    heapq.heapify(heap)
    res = []
    while heap and len(res) < k:
        _, i, j = heapq.heappop(heap)
        res.append([nums1[i], nums2[j]])
        if j + 1 < len(nums2):
            heapq.heappush(heap, (nums1[i] + nums2[j + 1], i, j + 1))
    return res
```

### 2. Add Two Numbers

Digits are stored in **reverse order** — least significant first. Walk both lists together, maintaining a carry. This is straightforward digit-by-digit addition.

**Prior knowledge:** Basic linked list traversal and dummy node construction.

```python
def addTwoNumbers(l1, l2):
    dummy = cur = ListNode(0)
    carry = 0
    while l1 or l2 or carry:
        val = carry
        if l1: val += l1.val; l1 = l1.next
        if l2: val += l2.val; l2 = l2.next
        carry, val = divmod(val, 10)
        cur.next = ListNode(val)
        cur = cur.next
    return dummy.next
```

### 445. Add Two Numbers II

Digits are stored in **forward order** — most significant first. Use **stacks** to reverse the access order, then add from least-significant digit up, building the result by prepending nodes.

**Prior knowledge:** [2. Add Two Numbers](https://leetcode.com/problems/add-two-numbers/) — same arithmetic logic, but reversed digit order. Also [206. Reverse Linked List](https://leetcode.com/problems/reverse-linked-list/) as an alternative approach.

```python
def addTwoNumbers(l1, l2):
    s1, s2 = [], []
    while l1: s1.append(l1.val); l1 = l1.next
    while l2: s2.append(l2.val); l2 = l2.next
    carry, head = 0, None
    while s1 or s2 or carry:
        val = carry
        if s1: val += s1.pop()
        if s2: val += s2.pop()
        carry, val = divmod(val, 10)
        node = ListNode(val)
        node.next = head
        head = node
    return head
```

### 287. Find the Duplicate Number

Array of n+1 integers in range [1, n] with one duplicate. Treat index-to-value as a linked list: `i -> nums[i]`. The duplicate creates a cycle. Apply **Floyd's cycle detection** to find the cycle entry — that's the duplicate value.

**Prior knowledge:** [142. Linked List Cycle II](https://leetcode.com/problems/linked-list-cycle-ii/) — the exact same Floyd's algorithm on an explicit linked list.

```python
def findDuplicate(nums):
    slow = fast = nums[0]
    while True:
        slow = nums[slow]
        fast = nums[nums[fast]]
        if slow == fast:
            break
    slow = nums[0]
    fast = nums[fast]       # keep fast at meeting point
    slow = 0                # reset to start
    while slow != fast:
        slow = nums[slow]
        fast = nums[fast]
    return slow
```

## Key Patterns to Remember

| Pattern | When to Use | Problems |
|---|---|---|
| Dummy node + prev skip | Remove entire groups of nodes | 82 |
| Min-heap k-way merge | Enumerate top-k from sorted streams | 373 |
| Carry-based digit addition | Sum numbers as linked lists | 2, 445 |
| Stack to reverse access order | Process forward-order list from tail | 445 |
| Floyd's on index-value graph | Find duplicate in bounded-range array | 287 |
