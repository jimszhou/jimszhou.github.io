Spent time reading through vLLM's scheduler and engine code to understand how continuous batching works in practice.

## Notes

- The `LLMEngine` orchestrates everything: tokenization, scheduling, model execution, and detokenization
- `Scheduler` manages three queues: waiting, running, and swapped
- Preemption policy: when GPU memory is exhausted, running sequences can be swapped to CPU memory
- The iteration-level scheduling is what enables continuous batching — each forward pass can serve a different set of sequences

Key file: `vllm/core/scheduler.py` — the heart of the system.
