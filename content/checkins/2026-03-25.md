## Core Technique: Linked List Two Pointers

Two pointers on linked lists come in several flavors: **fast-slow** (cycle detection, finding midpoint), **gap** (kth from end), and **parallel traversal** (merge, decompose, intersection). Combined with a **dummy head node** to simplify edge cases when building new lists, these cover nearly all singly linked list problems.

```python
# Dummy node template — use when building a new list
dummy = ListNode(-1)
p = dummy
# ... build list by assigning p.next ...
return dummy.next
```

**When to use it:** Whenever you see a singly linked list problem — merging, splitting, cycle detection, finding positions, or intersections.

**Why:** Avoids extra passes or O(n) space. Most solutions run in O(n) time and O(1) space.

## Problems Studied

### 21. Merge Two Sorted Lists

Use two pointers to compare heads of both lists, always appending the smaller node. Dummy node avoids null-check edge cases.

```python
def mergeTwoLists(l1, l2):
    dummy = p = ListNode(-1)
    while l1 and l2:
        if l1.val <= l2.val:
            p.next, l1 = l1, l1.next
        else:
            p.next, l2 = l2, l2.next
        p = p.next
    p.next = l1 or l2
    return dummy.next
```

### 86. Partition List

Split into two lists (< x and >= x) using two dummy heads, then connect them. Key gotcha: **sever each node's next pointer** from the original list to avoid creating cycles in the result.

```python
def partition(head, x):
    d1, d2 = ListNode(-1), ListNode(-1)
    p1, p2 = d1, d2
    while head:
        nxt = head.next
        head.next = None
        if head.val < x:
            p1.next, p1 = head, head
        else:
            p2.next, p2 = head, head
        head = nxt
    p1.next = d2.next
    return d1.next
```

### 23. Merge k Sorted Lists

Use a min-heap to always pick the smallest head among k lists. Each push/pop is O(log k), giving O(N log k) total.

```python
import heapq
def mergeKLists(lists):
    dummy = p = ListNode(-1)
    heap = [(h.val, i, h) for i, h in enumerate(lists) if h]
    heapq.heapify(heap)
    while heap:
        val, i, node = heapq.heappop(heap)
        p.next, p = node, node
        if node.next:
            heapq.heappush(heap, (node.next.val, i, node.next))
    return dummy.next
```

### 19. Remove Nth Node From End of List

Let `p1` advance `n+1` steps first, then move `p1` and `p2` together. When `p1` hits None, `p2` is right before the target. Dummy node handles the case where the head itself is removed.

```python
def removeNthFromEnd(head, n):
    dummy = ListNode(-1, head)
    p1, p2 = dummy, dummy
    for _ in range(n + 1):
        p1 = p1.next
    while p1:
        p1, p2 = p1.next, p2.next
    p2.next = p2.next.next
    return dummy.next
```

### 876. Middle of the Linked List

Slow moves 1 step, fast moves 2. When fast reaches the end, slow is at the middle. For even-length lists this returns the second middle node.

```python
def middleNode(head):
    slow = fast = head
    while fast and fast.next:
        slow, fast = slow.next, fast.next.next
    return slow
```

### 142. Linked List Cycle II

Fast-slow pointers meet inside the cycle. Then reset one to head and advance both at speed 1 — they meet at the cycle entry. Why: slow traveled k steps, fast traveled 2k; the extra k equals a full cycle multiple, so from the meeting point, walking k-m more steps reaches the entry — the same distance as head to entry.

```python
def detectCycle(head):
    slow = fast = head
    while fast and fast.next:
        slow, fast = slow.next, fast.next.next
        if slow == fast:
            slow = head
            while slow != fast:
                slow, fast = slow.next, fast.next
            return slow
    return None
```

### 160. Intersection of Two Linked Lists

Traverse A then B with p1, and B then A with p2. Both travel `lenA + lenB` steps total, so they arrive at the intersection node simultaneously. If no intersection, both reach None together.

```python
def getIntersectionNode(headA, headB):
    p1, p2 = headA, headB
    while p1 != p2:
        p1 = headB if p1 is None else p1.next
        p2 = headA if p2 is None else p2.next
    return p1
```

## Key Patterns to Remember

| Pattern | When to Use | Examples |
|---|---|---|
| Dummy head node | Building or splitting a new list | 21, 86, 23, 19 |
| Two-pointer merge | Combine sorted sequences | 21, 23 |
| Two-pointer decompose | Split list by condition | 86 |
| Sever node.next | Reattaching nodes to new lists (avoid cycles) | 86 |
| Fast-slow pointers | Find middle, detect cycle, find cycle entry | 876, 142 |
| Pointer gap of k | Find kth node from end | 19 |
| Cross-traverse (A+B / B+A) | Find intersection of two lists | 160 |
| Min-heap for k-way merge | Extend 2-list merge to k lists | 23 |
