## Core Technique: Binary Search

Binary search halves the search space each step, giving **O(log N)** time on sorted data. The idea is simple, but the details are devilish — should `mid` get +1 or -1? Should the while loop use `<=` or `<`?

The key to getting it right is understanding the **search interval**. This post uses the "closed both ends" convention `[left, right]` throughout, which keeps all three templates uniform and easy to remember.

**When to use:** Searching for a target (or its boundary) in a sorted array, or any problem where you can binary-partition a monotonic condition.

**Universal framework:**

```python
def binarySearch(nums: List[int], target: int) -> int:
    left, right = 0, len(nums) - 1  # closed interval [left, right]

    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] == target:
            ...           # depends on which template
        elif nums[mid] < target:
            left = mid + 1
        elif nums[mid] > target:
            right = mid - 1
    return ...
```

Three rules to remember:

1. **Write `else if`, not `else`.** Spell out every branch so the logic is explicit.
2. **`while left <= right`** — the loop stops when the interval is empty (`left == right + 1`), so no element is missed.
3. **`left = mid + 1` / `right = mid - 1`** — `mid` has been checked, remove it from the interval.

## Template 1: Find Exact Target

If the target exists, return its index; otherwise return -1. When `nums[mid] == target`, return immediately.

```python
def binary_search(nums: List[int], target: int) -> int:
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1
        elif nums[mid] > target:
            right = mid - 1
    return -1
```

**Limitation:** If `nums = [1,2,2,2,3]` and `target = 2`, this returns index 2 — but you can't control *which* 2 it finds. For that you need the boundary templates.

## Template 2: Find Left Boundary

Return the index of the **first** occurrence of `target` (leftmost). The trick: when `nums[mid] == target`, don't return — shrink the right boundary to keep searching left.

```python
def left_bound(nums: List[int], target: int) -> int:
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] < target:
            left = mid + 1
        elif nums[mid] > target:
            right = mid - 1
        elif nums[mid] == target:
            right = mid - 1  # shrink right boundary
    # when target doesn't exist, left is the index of
    # the smallest element greater than target
    if left < 0 or left >= len(nums):
        return -1
    return left if nums[left] == target else -1
```

**Why it works:** Setting `right = mid - 1` forces the search to continue in `[left, mid-1]`, effectively "locking on" to the leftmost match. When the loop exits, `left` points to the answer.

---

## Template 3: Find Right Boundary

Return the index of the **last** occurrence of `target` (rightmost). Mirror of template 2: when `nums[mid] == target`, shrink the left boundary to keep searching right.

```python
def right_bound(nums: List[int], target: int) -> int:
    left, right = 0, len(nums) - 1
    while left <= right:
        mid = left + (right - left) // 2
        if nums[mid] < target:
            left = mid + 1
        elif nums[mid] > target:
            right = mid - 1
        elif nums[mid] == target:
            left = mid + 1   # shrink left boundary
    # when target doesn't exist, right is the index of
    # the largest element smaller than target
    if right < 0 or right >= len(nums):
        return -1
    return right if nums[right] == target else -1
```

**Why it works:** Setting `left = mid + 1` forces the search into `[mid+1, right]`, locking on to the rightmost match. When the loop exits, `right` points to the answer.

## Problems Studied

### 704. Binary Search

Given a sorted array `nums` and a `target`, return the index of `target` or -1. Straightforward application of Template 1.

```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:
        left, right = 0, len(nums) - 1
        while left <= right:
            mid = left + (right - left) // 2
            if nums[mid] == target:
                return mid
            elif nums[mid] < target:
                left = mid + 1
            elif nums[mid] > target:
                right = mid - 1
        return -1
```

### 34. Find First and Last Position of Element in Sorted Array

Given a sorted array, find the starting and ending position of `target`. This is exactly "find left boundary + find right boundary" — Templates 2 and 3 combined.

```python
class Solution:
    def searchRange(self, nums: List[int], target: int) -> List[int]:
        return [self.left_bound(nums, target),
                self.right_bound(nums, target)]

    def left_bound(self, nums, target):
        left, right = 0, len(nums) - 1
        while left <= right:
            mid = left + (right - left) // 2
            if nums[mid] < target:
                left = mid + 1
            elif nums[mid] > target:
                right = mid - 1
            elif nums[mid] == target:
                right = mid - 1
        if left < 0 or left >= len(nums):
            return -1
        return left if nums[left] == target else -1

    def right_bound(self, nums, target):
        left, right = 0, len(nums) - 1
        while left <= right:
            mid = left + (right - left) // 2
            if nums[mid] < target:
                left = mid + 1
            elif nums[mid] > target:
                right = mid - 1
            elif nums[mid] == target:
                left = mid + 1
        if right < 0 or right >= len(nums):
            return -1
        return right if nums[right] == target else -1
```

## Key Patterns to Remember

| Template | `nums[mid] == target` | Returns | Use Case |
|---|---|---|---|
| Find exact | `return mid` | Any matching index | LC 704 — simple lookup |
| Left boundary | `right = mid - 1` | `left` (first match) | LC 34 — first position |
| Right boundary | `left = mid + 1` | `right` (last match) | LC 34 — last position |

The only difference between the three templates is **what happens when `nums[mid] == target`** and **which pointer to return**. Everything else — the `while left <= right` condition, the `mid ± 1` updates, the closed interval — stays exactly the same.
