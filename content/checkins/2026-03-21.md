Spent the evening studying the [PagedAttention paper](https://arxiv.org/abs/2309.06180) (Kwon et al., 2023) — the core innovation behind vLLM's memory management.

## Key Takeaways

**The Problem**: Traditional LLM serving pre-allocates contiguous KV cache memory for each request based on max sequence length. This leads to massive internal fragmentation (60-80% waste) because most sequences don't reach max length.

**The Solution**: PagedAttention borrows virtual memory concepts from OS design:
- KV cache is divided into fixed-size **blocks** (like memory pages)
- Blocks are allocated on-demand as tokens are generated
- A **block table** maps logical blocks to physical blocks (like a page table)
- Non-contiguous physical memory can back contiguous logical sequences

**Why It Matters**: This simple insight increases serving throughput by 2-4x compared to FasterTransformer and HuggingFace TGI, purely through better memory management.
