## Core Technique: Binary Search on Answer

Yesterday we covered binary search on sorted arrays. But most real problems don't hand you a sorted array — instead, you need to **search over possible answers**. The key insight: if there's a **monotonic relationship** between your answer and some constraint, you can binary search the answer space.

**The three-step framework:**

1. **Identify x** — what the problem asks you to minimize/maximize. This is the search variable.
2. **Define f(x)** — a function of x that is **monotonic**. Usually maps x to a cost, count, or time.
3. **Identify target** — the constraint on f(x) given by the problem.

Once you have these three, the problem reduces to: *find the smallest (or largest) x such that f(x) meets target.*

**Generic template:**

```python
def binary_search_on_answer(lo, hi, target):
    # lo, hi = bounds of x's possible values
    while lo <= hi:
        mid = lo + (hi - lo) // 2
        if f(mid) <= target:   # f is decreasing; flip for increasing
            hi = mid - 1       # mid works → try smaller x (left boundary)
        else:
            lo = mid + 1
    return lo
```

**How to find x, f(x), target in a problem:**
- **x**: whatever the problem says "return the minimum/maximum of" — that's your search variable.
- **f(x)**: simulate the problem with a fixed x, and compute the resulting cost/time/count. Check that f is monotonic in x.
- **target**: the constraint the problem gives you (time limit, number of days, number of splits).

## Problems Studied

### 875. Koko Eating Bananas

**Mapping:** x = eating speed, f(x) = hours to finish all piles (monotonically decreasing), target = h. We want the smallest x where f(x) ≤ h — left boundary search.

```python
class Solution:
    def minEatingSpeed(self, piles: List[int], h: int) -> int:
        def hours_needed(speed):
            return sum((p + speed - 1) // speed for p in piles)

        lo, hi = 1, max(piles)
        while lo <= hi:
            mid = lo + (hi - lo) // 2
            if hours_needed(mid) <= h:
                hi = mid - 1
            else:
                lo = mid + 1
        return lo
```

### 1011. Capacity To Ship Packages Within D Days

**Mapping:** x = ship capacity, f(x) = days needed to ship all packages (monotonically decreasing), target = days. Smallest x where f(x) ≤ days.

```python
class Solution:
    def shipWithinDays(self, weights: List[int], days: int) -> int:
        def days_needed(cap):
            d, cur = 1, 0
            for w in weights:
                if cur + w > cap:
                    d += 1
                    cur = 0
                cur += w
            return d

        lo, hi = max(weights), sum(weights)
        while lo <= hi:
            mid = lo + (hi - lo) // 2
            if days_needed(mid) <= days:
                hi = mid - 1
            else:
                lo = mid + 1
        return lo
```

### 410. Split Array Largest Sum

**Key insight:** this is *the same problem* as 1011. "Split into m subarrays minimizing the max sum" = "ship all packages in m days with minimum capacity." x = max subarray sum allowed, f(x) = number of splits needed, target = m.

```python
class Solution:
    def splitArray(self, nums: List[int], k: int) -> int:
        def splits_needed(max_sum):
            count, cur = 1, 0
            for n in nums:
                if cur + n > max_sum:
                    count += 1
                    cur = 0
                cur += n
            return count

        lo, hi = max(nums), sum(nums)
        while lo <= hi:
            mid = lo + (hi - lo) // 2
            if splits_needed(mid) <= k:
                hi = mid - 1
            else:
                lo = mid + 1
        return lo
```

## Key Patterns to Remember

| Problem | x (search var) | f(x) | target | Monotonicity |
|---|---|---|---|---|
| 875 Koko Bananas | eating speed | hours to finish | h | f decreasing |
| 1011 Ship Packages | ship capacity | days to ship | days | f decreasing |
| 410 Split Array | max subarray sum | # of splits | k | f decreasing |

**Universal recipe:** problem asks "minimize x subject to constraint" → binary search x, check feasibility with f(x), search left boundary. All three problems above follow this exact pattern — only f(x) changes.
