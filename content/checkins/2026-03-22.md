## Core Technique: Prefix Sum

The idea is simple: precompute cumulative sums so that any subarray sum `nums[i..j]` becomes `prefix[j+1] - prefix[i]` in O(1). This turns brute-force O(n²) subarray problems into O(n) solutions. Combined with a hash map to track previously seen prefix values, it unlocks a huge class of problems.

```python
# Basic prefix sum template
prefix = [0] * (len(nums) + 1)
for i in range(len(nums)):
    prefix[i + 1] = prefix[i] + nums[i]

# sum of nums[i..j] = prefix[j+1] - prefix[i]
```

**When to use it:** any time you need subarray sums, subarray counts, or need to detect a target sum within a contiguous range.

## Problems Studied

### 724. Find Pivot Index
Find the index where left sum equals right sum. Compute total sum first, then scan left-to-right tracking the running left sum. If `left_sum == total - left_sum - nums[i]`, that's the pivot.

### 238. Product of Array Except Self
Prefix sum's multiplicative cousin. Build a `left[]` product array and a `right[]` product array, then `answer[i] = left[i] * right[i]`. Can be done in O(1) extra space by accumulating the right product in a single variable on the second pass.

### 560. Subarray Sum Equals K
The classic prefix sum + hash map combo. Store counts of each prefix sum seen so far. At each index, check if `prefix_sum - k` exists in the map — if so, there are that many subarrays ending here with sum `k`.

### 523. Continuous Subarray Sum
Similar to 560 but we care about sums that are **multiples of k**. Store `prefix_sum % k` in a hash map. If the same remainder appears again and the subarray length is ≥ 2, we found a valid subarray (because the difference between two equal remainders is divisible by k).

### 525. Contiguous Array
Transform 0s to -1s, then it becomes "find the longest subarray with sum 0." Use a hash map to record the first index where each prefix sum occurs. When the same prefix sum appears again, the subarray between those indices has equal 0s and 1s.

### 1314. Matrix Block Sum
2D extension of prefix sum. Build a 2D prefix sum matrix where `prefix[i][j]` = sum of all elements in the rectangle `(0,0)` to `(i-1,j-1)`. Then any rectangular region sum is computed with inclusion-exclusion in O(1).

### 1352. Product of the Last K Numbers
Maintain a running product prefix array. On `add(num)`, append `prefix[-1] * num`. On `getProduct(k)`, return `prefix[-1] / prefix[-k-1]`. Reset the prefix when a zero is added since it wipes out all prior products.

### 1124. Longest Well-Performing Interval
Map each day to +1 (hours > 8) or -1 (hours ≤ 8). Then find the longest subarray with sum > 0. Use a prefix sum and a hash map storing the first occurrence of each prefix value. If `prefix_sum > 0`, the whole array so far works. Otherwise, look for the first index where prefix sum was `prefix_sum - 1`.

## Key Patterns to Remember

| Pattern | When to Use | Examples |
|---|---|---|
| Prefix sum + hash map | Count/find subarrays with target sum | 560, 523, 525, 1124 |
| Prefix product | Multiplicative queries | 238, 1352 |
| 2D prefix sum | Rectangle region queries | 1314 |
| Prefix sum + modular arithmetic | Divisibility conditions | 523 |
